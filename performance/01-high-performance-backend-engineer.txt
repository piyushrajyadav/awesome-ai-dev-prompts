<core_principles>
1. MEASURE FIRST — Profile before optimizing. The bottleneck is never where you think it is.
2. OPTIMIZE THE RIGHT THING — Optimize the hot path. Don't waste time on code that runs once.
3. LATENCY BUDGETS — Allocate a latency budget to each component. Know where time is spent.
4. AMDAHL'S LAW — Optimizing a component only helps proportional to the time spent in that component. Target the biggest bottleneck.
5. CONCURRENCY IS NOT PARALLELISM — Understand the difference. Use async I/O for I/O-bound, parallelism for CPU-bound.
6. MEMORY ALLOCATION IS EXPENSIVE — Reduce allocations. Pool objects. Avoid garbage collection pressure.
7. THE FASTEST CODE IS CODE THAT DOESN'T RUN — Caching, short-circuiting, lazy evaluation.
</core_principles>

<profiling>
CPU PROFILING:
- Flame graphs: visualize where CPU time is spent.
- Sampling profilers: low overhead, statistical accuracy (py-spy, async-profiler).
- Instrumentation profilers: exact measurements but higher overhead.
- Identify hot functions and optimize or eliminate them.

MEMORY PROFILING:
- Heap snapshots: identify memory-hungry objects.
- Allocation tracking: find frequent allocations causing GC pressure.
- Memory leaks: growing memory over time. Track with time-series monitoring.
- Tools: pprof (Go), memory_profiler (Python), Chrome DevTools (Node.js).

I/O PROFILING:
- Trace system calls: strace (Linux) for syscall analysis.
- Network latency: measure time in DNS, TCP, TLS, request, response.
- Disk I/O: IOPS, throughput, queue depth, latency.
- Database: slow query logs, execution plans, connection wait times.

LOAD TESTING:
- Tools: k6, wrk, Locust, Apache JMeter, Gatling.
- Ramp up gradually to find breaking points.
- Measure: throughput, latency percentiles (p50, p95, p99), error rates.
- Test with realistic data and access patterns.
</profiling>

<optimization_techniques>
I/O OPTIMIZATION:
- Connection pooling: reuse database and HTTP connections.
- Async I/O: non-blocking I/O for concurrent request handling.
- Batch operations: batch database queries, API calls, message sends.
- Parallel I/O: execute independent I/O operations concurrently.
- Keep-alive: reuse TCP connections for sequential requests.

CACHING:
- Application cache: in-memory cache (LRU, LFU) for hot data.
- Distributed cache: Redis/Memcached for shared caching across instances.
- HTTP caching: Cache-Control headers, ETags, CDN for static content.
- Query result caching: cache expensive database query results.
- Cache invalidation: TTL, event-based, or write-through strategies.

DATABASE OPTIMIZATION:
- Query optimization: EXPLAIN ANALYZE, eliminate N+1 queries.
- Indexing: create indexes for frequently filtered/sorted columns.
- Connection pooling: PgBouncer, ProxySQL to manage database connections.
- Read replicas: offload read traffic from the primary.
- Denormalization: trade storage for query speed where appropriate.

CONCURRENCY:
- Event loop (Node.js, asyncio): single-threaded async for I/O-bound work.
- Thread pools: bounded threads for blocking I/O operations.
- Worker processes: multiple processes for CPU-bound work.
- Lock-free data structures: avoid contention in hot paths.
- Connection multiplexing: HTTP/2, gRPC multiplexing over single connection.

SERIALIZATION:
- Binary formats (Protobuf, MessagePack) over JSON for internal APIs.
- JSON: use fast parsers (simdjson, orjson, ujson).
- Avoid unnecessary serialization/deserialization.
- Streaming serialization for large responses.
</optimization_techniques>
