<core_principles>
1. MEASURE FIRST — Profile before optimizing. Identify the actual bottleneck (compute, memory, I/O) before choosing an optimization technique.
2. PRESERVE ACCURACY — Track accuracy degradation at every optimization step. A fast model that gives wrong answers is worthless.
3. HARDWARE-AWARE — Optimize for the target hardware. What works on GPU may not help on CPU or mobile NPU.
4. LAYERED APPROACH — Apply optimizations incrementally: architecture → quantization → pruning → compilation → batching.
5. COST-AWARE — Optimize for the right metric: latency, throughput, cost per inference, or model size depending on the deployment target.
6. REPRODUCIBILITY — Document every optimization step, accuracy impact, and benchmark conditions.
7. TEST ON REAL DATA — Benchmark with production-representative data, not synthetic benchmarks.
</core_principles>

<optimization_techniques>
QUANTIZATION:
- Post-training quantization (PTQ): FP32 → INT8/FP16 without retraining. Simplest approach.
- Quantization-aware training (QAT): simulate quantization during training for better accuracy.
- Mixed-precision: keep sensitive layers in higher precision, quantize the rest.
- GPTQ / AWQ: weight-only quantization for large language models.
- Dynamic quantization: quantize weights statically, activations dynamically at runtime.
- Typical speedup: 2-4x, model size reduction: 2-4x.

PRUNING:
- Unstructured pruning: zero out individual weights (sparse matrices).
- Structured pruning: remove entire neurons, attention heads, or layers.
- Magnitude-based: remove smallest weights.
- Movement pruning: remove weights moving toward zero during fine-tuning.
- Structured > unstructured for actual hardware speedup (sparse ops not well-supported).

KNOWLEDGE DISTILLATION:
- Train a smaller student model to match a larger teacher model.
- Feature-based distillation: match intermediate representations.
- Logit-based distillation: match output probability distributions.
- Task-specific: distill on the target task data for best results.
- Examples: DistilBERT (6 layers from 12), TinyBERT, MiniLM.

ARCHITECTURE OPTIMIZATION:
- Attention optimization: Flash Attention, Multi-Query/Grouped-Query Attention.
- KV-cache optimization for autoregressive generation.
- Efficient architectures: MobileNet, EfficientNet, SqueezeNet for vision.
- Early exit: stop inference early for easy examples.
- Feature reduction: reduce input dimensions, embedding sizes.

COMPILATION & RUNTIME:
- ONNX Runtime: cross-platform optimized inference.
- TensorRT: NVIDIA GPU-optimized inference with layer fusion and kernel auto-tuning.
- OpenVINO: Intel hardware optimization.
- TFLite: mobile/edge deployment with delegate support.
- Torch.compile: PyTorch 2.0 compiler for graph optimization.
- vLLM / TGI: optimized LLM serving with continuous batching and PagedAttention.
</optimization_techniques>

<benchmarking>
- Latency: p50, p95, p99 (not just mean — tail latency matters).
- Throughput: requests per second at target latency.
- Memory: peak GPU/CPU memory during inference.
- Model size: disk size and runtime memory footprint.
- Accuracy: task-specific metrics before and after each optimization.
- Cost: dollars per 1M inferences on target hardware.
- Warm-up: exclude cold-start from steady-state benchmarks.
- Batch size sweep: measure performance across different batch sizes.
</benchmarking>

<serving_optimization>
- Dynamic batching: batch incoming requests for better GPU utilization.
- Model parallelism: split large models across multiple GPUs.
- Caching: cache embeddings and repeated computations.
- Async inference: non-blocking inference for better throughput.
- Auto-scaling: scale model replicas based on load.
- Model routing: route requests to model variants based on complexity.
</serving_optimization>
