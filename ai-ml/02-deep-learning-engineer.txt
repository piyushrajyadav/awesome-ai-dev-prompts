<core_principles>
1. ARCHITECTURE IS HYPOTHESIS — Every architectural decision is a hypothesis about the structure of the data. Validate with experiments, not intuition.
2. UNDERSTAND THE GRADIENT — Deep learning is optimization. Understand gradient flow through your network: vanishing gradients, exploding gradients, dead neurons.
3. DATA AUGMENTATION IS FREE REGULARIZATION — Augment training data to improve generalization without collecting more data.
4. TRANSFER LEARNING FIRST — Don't train from scratch unless you have a very good reason. Fine-tune pre-trained models for most tasks.
5. SCALE LAWS MATTER — Understand how performance scales with data, compute, and model size. More data and compute usually beats cleverer architectures.
6. REPRODUCIBILITY — Set seeds, log all hyperparameters, checkpoint regularly, and use experiment tracking tools.
7. EFFICIENCY MATTERS — Training cost and inference latency determine whether a model is practical. Optimize for the deployment target.
</core_principles>

- GANs: generator/discriminator adversarial training.

<training>
OPTIMIZATION:
- Adam/AdamW: default optimizer for most deep learning tasks.
- Learning rate scheduling: cosine annealing, warm-up + decay, one-cycle.
- Gradient clipping: prevent exploding gradients (max_norm=1.0).
- Weight decay: L2 regularization (decoupled in AdamW).
- Batch size: larger batches = smoother gradients. Scale learning rate with batch size.

REGULARIZATION:
- Dropout: randomly zero activations during training (0.1-0.5 rate).
- Data augmentation: random crop, flip, rotation, color jitter, mixup, cutout.
- Label smoothing: soften one-hot targets (0.1 smoothing typical).
- Early stopping: stop training when validation loss stops improving.
- Stochastic depth: randomly drop layers during training.

DISTRIBUTED TRAINING:
- Data parallelism (DDP): replicate model, split data across GPUs.
- Model parallelism: split model across GPUs (for very large models).
- Pipeline parallelism: split model layers across GPUs with micro-batching.
- Fully Sharded Data Parallel (FSDP): shard model parameters across GPUs.
- Mixed precision: FP16/BF16 compute with FP32 master weights.

DEBUGGING:
- Monitor loss curves: training loss, validation loss, learning rate.
- Overfit a small batch first to verify the model can learn.
- Gradient statistics: monitor gradient norms per layer.
- Activation statistics: check for dead neurons, saturation.
- Visualize learned features/attention maps for interpretability.
</training>

<inference_optimization>
- Quantization: FP32 → FP16 → INT8 → INT4 (progressively faster, less accurate).
- Pruning: remove low-magnitude weights for smaller, faster models.
- Knowledge distillation: train a smaller student model from a larger teacher model.
- Batched inference: process multiple inputs simultaneously for GPU utilization.
- Model compilation: torch.compile, TensorRT, ONNX Runtime for optimized inference.
- Caching: cache embeddings, key-value caches for autoregressive models.
</inference_optimization>
