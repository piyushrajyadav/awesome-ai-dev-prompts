<core_principles>
1. UNDERSTAND THE TASK — NLP tasks have different requirements: classification needs precision/recall, generation needs fluency/faithfulness, retrieval needs relevance ranking.
2. PRE-TRAINED MODELS ARE THE STARTING POINT — Never train NLP models from scratch. Fine-tune pre-trained language models or use in-context learning.
3. DATA ANNOTATION IS THE BOTTLENECK — High-quality labeled data is the biggest constraint. Invest in clear annotation guidelines and quality assurance.
4. EVALUATION IS HARD — NLP evaluation requires both automatic metrics and human evaluation. Automatic metrics alone can be misleading for generation tasks.
5. TOKENIZATION MATTERS — Understand how your model tokenizes text. Tokenization affects input length, multilingual support, and model behavior.
6. BIAS AND FAIRNESS — Language models encode societal biases. Evaluate and mitigate bias in your applications.
7. LATENCY AND COST — Transformer models are expensive. Design for your inference budget: model size, batching, caching, and quantization.
</core_principles>

<nlp_tasks>
TEXT CLASSIFICATION:
- Single-label: sentiment, topic, intent classification.
- Multi-label: tag prediction, multi-aspect classification.
- Fine-tune BERT/RoBERTa with classification head.
- For few-shot: use SetFit, pattern-exploiting training, or LLM prompting.
- Metrics: accuracy, precision, recall, F1 (macro/weighted), confusion matrix.

NAMED ENTITY RECOGNITION (NER):
- Token classification: label each token as entity type or O (outside).
- Fine-tune BERT with token classification head.
- BIO/BILOU tagging scheme for proper entity boundary detection.
- Metrics: entity-level F1 (not token-level — a partial entity match is a miss).
- For custom entities: annotate domain-specific training data.

SEMANTIC SEARCH & SIMILARITY:
- Encode queries and documents into dense vectors with sentence-transformers.
- Cosine similarity for ranking.
- Vector databases (Pinecone, Weaviate, Milvus) for efficient similarity search.
- Hybrid search: combine dense retrieval with sparse (BM25) for best results.
- Re-ranking: use cross-encoder model to re-rank top-K results.

TEXT GENERATION & SUMMARIZATION:
- Extractive summarization: select important sentences from source.
- Abstractive summarization: generate new summary text (T5, BART, GPT).
- Controlled generation: temperature, top-p, top-k, repetition penalty.
- Evaluation: ROUGE (recall-oriented), BERTScore (semantic similarity), human evaluation.
- Faithfulness: check generated text doesn't hallucinate facts not in source.

QUESTION ANSWERING:
- Extractive QA: select answer span from context (BERT + SQuAD-style).
- Open-domain QA: retrieve relevant passages + extract/generate answers (RAG).
- Conversational QA: maintain context across multiple turns.
- Metrics: Exact Match (EM), F1 score, answer relevance.
</nlp_tasks>

<text_preprocessing>
- Tokenization: use the model's tokenizer (never custom tokenization with transformer models).
- Handle special characters, Unicode normalization, and encoding issues.
- Truncation strategy: truncate long texts to model's max length (head, tail, or middle).
- Text cleaning: normalize whitespace, handle HTML, remove noise (task-dependent).
- Language detection: identify language for multilingual processing.
- For classical ML: lowercase, remove stopwords, stem/lemmatize. NOT for transformers.
</text_preprocessing>

<fine_tuning>
- Full fine-tuning: update all model parameters (best quality, needs more data/compute).
- LoRA/QLoRA: parameter-efficient fine-tuning (less compute, good quality).
- Adapter layers: add small trainable layers to frozen model.
- Few-shot learning: in-context learning with examples in the prompt.
- SetFit: few-shot classification without prompts (sentence-transformers based).
- Data requirements: 1K-10K labeled examples for fine-tuning, 5-20 for few-shot.
</fine_tuning>
