<core_principles>
1. ASYNC EVERYTHING — Node.js is single-threaded. Never block the event loop. All I/O must be asynchronous.
2. TYPESCRIPT IS MANDATORY — Untyped Node.js is a maintenance nightmare. Use strict TypeScript for all server-side code.
3. ERROR HANDLING IS NOT OPTIONAL — Unhandled promise rejections crash Node.js processes. Every async operation must have proper error handling.
4. STREAMS FOR LARGE DATA — Never load large files or datasets into memory. Use streams (Readable, Writable, Transform, Duplex).
5. DEFEND AGAINST THE EVENT LOOP — CPU-intensive work must be offloaded to Worker Threads, child processes, or external services.
6. DEPENDENCY DISCIPLINE — Node.js ecosystem has millions of packages, most poorly maintained. Vet dependencies carefully. Prefer well-maintained, minimal packages.
7. STRUCTURED APPLICATION — Node.js flexibility is a trap. Enforce clear architecture: routes → controllers → services → repositories.
</core_principles>

<async_patterns>
PROMISES & ASYNC/AWAIT:
- Always use async/await over raw promises.
- Never mix callbacks and promises. Promisify callback APIs with util.promisify.
- Use Promise.all for independent parallel operations.
- Use Promise.allSettled when you need all results regardless of failures.
- Always await or catch promises. Unhandled rejections crash the process in Node.js 15+.

EVENT LOOP AWARENESS:
- Understand the event loop phases: timers → pending → idle → poll → check → close.
- process.nextTick runs before any I/O. setImmediate runs after I/O.
- Long-running synchronous code blocks the event loop — split with setImmediate or Worker Threads.
- Monitor event loop lag: if > 100ms, you have a blocking problem.

STREAMS:
- Use streams for file processing, HTTP request/response bodies, and data transformation.
- Pipeline: use stream.pipeline() (not .pipe()) for proper error handling.
- Backpressure: respect writable.write() return value and 'drain' event.
- Transform streams for data processing pipelines (CSV parsing, JSON transformation).
</async_patterns>

<performance>
- Use clustering (PM2, Node.js cluster module) to utilize all CPU cores.
- Connection pooling for database, Redis, and HTTP clients.
- Implement proper caching: in-memory (LRU cache), Redis for shared cache.
- Avoid synchronous file system and crypto operations (use async variants).
- Use Buffer pooling for high-throughput binary processing.
- Profile with --inspect and Chrome DevTools, clinic.js, or 0x for flamegraphs.
- Monitor memory usage: watch for heap growth indicating memory leaks.
- Use WeakRef and FinalizationRegistry for cache entries that should be GC'd.
</performance>
