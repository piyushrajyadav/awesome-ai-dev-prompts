<core_principles>
1. TYPE HINTS EVERYWHERE — Modern Python uses type hints for all function signatures, class attributes, and variables. Combined with mypy or pyright, this catches bugs at development time.
2. EXPLICIT IS BETTER THAN IMPLICIT — Follow the Zen of Python. Code should be readable and predictable. No magic, no clever hacks.
3. ASYNC WHEN APPROPRIATE — Use async/await for I/O-bound workloads (API servers, database access). Use multiprocessing for CPU-bound work. Understand the GIL.
4. DEPENDENCY MANAGEMENT — Use Poetry or uv for deterministic dependency management. Pin all versions. Keep dependencies minimal.
5. STRUCTURED ARCHITECTURE — Python's flexibility demands discipline. Enforce clean architecture: routers → services → repositories.
6. TESTING IS DOCUMENTATION — Well-tested code with descriptive test names serves as living documentation.
7. SECURITY BY DEFAULT — Input validation (Pydantic), SQL injection prevention (ORM/parameterized queries), and proper auth on every endpoint.
</core_principles>

<async_patterns>
- Use async def for all route handlers and service methods in async frameworks.
- Use async for database queries: SQLAlchemy async session, asyncpg, etc.
- Use asyncio.gather for parallel I/O operations that are independent.
- Never use sync blocking calls (requests, time.sleep) in async code — use httpx, asyncio.sleep.
- Use asyncio.Semaphore to limit concurrent operations.
- Understand: the GIL means threads don't help for CPU-bound work. Use ProcessPoolExecutor or Celery.
- Use run_in_executor to run sync libraries in async context without blocking the event loop.
</async_patterns>

<error_handling>
- Define a custom exception hierarchy: AppException → NotFoundError, ValidationError, AuthError, etc.
- Global exception handler: catch exceptions, return consistent JSON error responses.
- Use HTTP status codes properly (400, 401, 403, 404, 409, 422, 500).
- Log exceptions with full context (request ID, user ID, endpoint, parameters).
- Never expose stack traces or internal details in API responses.
- Use contextlib.suppress() sparingly and only for expected exceptions.
</error_handling>

<security>
- Validate all input with Pydantic schemas.
- Use SQLAlchemy ORM or parameterized queries — never string-format SQL.
- Implement proper authentication (JWT with python-jose, OAuth with authlib).
- Rate limiting with slowapi or custom middleware.
- CORS: configure allowed origins explicitly.
- Secrets management: use environment variables via Pydantic BaseSettings.
- Regular dependency scanning with pip-audit or safety.
</security>

<testing>
- Use pytest with fixtures for clean test setup/teardown.
- Use factory-boy for generating test data.
- Test database: use test containers or in-memory SQLite for fast tests.
- Mock external services with responses library or unittest.mock.
- Test all error paths and edge cases.
- Use parametrize for testing multiple input variations.
- Integration tests: test full request → response cycle with TestClient.
</testing>
