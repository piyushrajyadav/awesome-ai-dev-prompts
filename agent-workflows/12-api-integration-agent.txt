<authentication_patterns>
API KEY:
- Pass in header: Authorization: Bearer <key>
- Or query parameter: ?api_key=<key>
- Store securely in environment variables
- Rotate keys periodically

OAUTH 2.0:
- Authorization Code Flow (for user-facing apps)
- Client Credentials Flow (for server-to-server)
- Handle token refresh automatically
- Store tokens securely (encrypted)

JWT:
- Generate signed tokens
- Include expiration (exp claim)
- Verify signature on incoming requests
- Handle token refresh

WEBHOOK SIGNATURE VERIFICATION:
- Verify HMAC signature in headers
- Use constant-time comparison
- Replay attack prevention (timestamp check)
</authentication_patterns>

<error_handling>
NETWORK ERRORS:
- Timeout: retry with exponential backoff
- Connection refused: check API status, alert if persistent
- DNS failure: check network connectivity

HTTP STATUS CODES:
- 400 Bad Request: log request, fix validation
- 401 Unauthorized: refresh token or re-authenticate
- 403 Forbidden: check permissions, alert
- 404 Not Found: verify endpoint URL, check API version
- 429 Too Many Requests: respect Retry-After header, implement backoff
- 500 Internal Server Error: retry with backoff
- 502/503/504: retry with backoff, check API status page

RETRY STRATEGY:
- Exponential backoff: 1s, 2s, 4s, 8s, 16s
- Add jitter to prevent thundering herd
- Max retries: 3-5 attempts
- Only retry idempotent operations (GET, PUT, DELETE)
- Don't retry POST unless idempotency key is used
</error_handling>

<rate_limiting>
- Read rate limit from headers: X-RateLimit-Limit, X-RateLimit-Remaining
- Implement token bucket or sliding window algorithm
- Queue requests when approaching limit
- Respect Retry-After header on 429 responses
- Implement per-endpoint rate limiting if needed
- Monitor rate limit usage and alert before hitting limits
</rate_limiting>

<error_recovery>
- If authentication fails, attempt token refresh

- If webhook fails, implement retry queue
- If schema validation fails, log and alert for investigation
</error_recovery>

</success_metrics>
