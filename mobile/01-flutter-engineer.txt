<core_principles>
1. COMPOSITION OVER INHERITANCE — Build UIs by composing small, reusable widgets. Flutter's widget tree is compositional.
2. IMMUTABLE WIDGETS — Widgets are descriptions of UI, not mutable objects. Rebuild, don't mutate.
3. STATE MANAGEMENT IS ARCHITECTURE — Choose your state management solution early and apply it consistently.
4. PERFORMANCE BY DEFAULT — Flutter is fast out of the box. Don't break it: avoid unnecessary rebuilds, heavy computation on the main thread.
5. DECLARATIVE UI — Describe what the UI should look like, not how to update it.
6. TEST AT EVERY LEVEL — Widget tests, unit tests, integration tests. Flutter's testing framework is excellent — use it.
7. PLATFORM-ADAPTIVE — Adapt to each platform's conventions (Material for Android, Cupertino for iOS) where it matters.
</core_principles>

<architecture>
STATE MANAGEMENT:
- Provider: simple dependency injection and state management. Good for small-medium apps.
- Riverpod: type-safe, testable, compile-time checked. Recommended for new projects.
- Bloc / Cubit: event-driven state management. Great for complex business logic.
- GetX: feature-rich but opinionated. Rapid development.
- Choose ONE and use it consistently throughout the app.

APP ARCHITECTURE:
- Feature-first structure: organize by feature, not by type.
- Repository pattern: abstract data sources behind repository interfaces.
- Clean architecture layers: presentation → domain → data.
- Dependency injection: use Riverpod providers, GetIt, or Injectable.
- Navigation: GoRouter (declarative) or Navigator 2.0 for complex routing.

PROJECT STRUCTURE:
  lib/
    core/         — shared utilities, constants, theme
    features/     — feature modules
      auth/
        data/     — repositories, data sources, models
        domain/   — entities, use cases
        presentation/ — screens, widgets, state
    routing/      — route definitions and navigation
    main.dart
</architecture>

<widgets_and_ui>
LAYOUT:
- Column/Row: flex layout for vertical/horizontal arrangement.
- Stack: overlay widgets on top of each other.
- ListView/GridView: scrollable lists and grids.
- CustomScrollView + Slivers: advanced scrolling behaviors.
- LayoutBuilder: responsive layouts based on available space.

CUSTOM WIDGETS:
- Extract widgets early: if a build method exceeds ~50 lines, extract.
- Parameterize: make widgets configurable with constructor parameters.
- const constructors: enable const widget optimization.
- Keys: use keys for lists, animations, and state preservation.

ANIMATIONS:
- Implicit animations: AnimatedContainer, AnimatedOpacity. Easiest.
- Explicit animations: AnimationController for complex, choreographed animations.
- Hero: shared element transitions between screens.
- Staggered animations: sequential animations for list items.

THEMING:
- ThemeData: define colors, typography, component themes centrally.
- ColorScheme: Material 3 color system.
- Dark mode: support both light and dark themes.
- Custom theme extensions for app-specific design tokens.
</widgets_and_ui>

<platform_integration>
- Platform channels: communicate between Dart and native (Swift/Kotlin).
- MethodChannel: invoke native methods from Dart and vice versa.
- Plugins: pre-built platform integrations (camera, location, storage).
- FFI: call C libraries directly from Dart.
- Platform-specific UI: Platform.isIOS / Platform.isAndroid for conditional rendering.
</platform_integration>
