<core_principles>
1. JETPACK COMPOSE FIRST — Use Compose for new UI. It's declarative, testable, and the future of Android UI.
2. KOTLIN IDIOMS — Use Kotlin features: coroutines, flows, sealed classes, extension functions. Don't write Java in Kotlin.
3. LIFECYCLE-AWARE — Respect Android lifecycle. Use lifecycle-aware components to prevent leaks and crashes.
4. UNIDIRECTIONAL DATA FLOW — State flows down, events flow up. ViewModel exposes state, UI dispatches events.
5. OFFLINE-FIRST — Cache data locally. Android users have unreliable networks. Room + DataStore.
6. BACKGROUND WORK — Use WorkManager for guaranteed background execution. Don't fight the system.
7. DEVICE FRAGMENTATION — Test on multiple screen sizes, Android versions, and manufacturers.
</core_principles>

<architecture>
APP ARCHITECTURE (GOOGLE RECOMMENDED):
- UI Layer: Compose screens and ViewModels.
- Domain Layer (optional): use cases for complex business logic.
- Data Layer: repositories, data sources (network, database, preferences).
- Dependency Injection: Hilt (built on Dagger, recommended by Google).

VIEWMODEL + STATE:
- ViewModel survives configuration changes (rotation).
- StateFlow/SharedFlow for reactive state emission.
- UI State: single sealed class/data class representing screen state.
- Events: sealed class for user actions dispatched to ViewModel.
- Side effects: one-time events (navigation, snackbar) via SharedFlow/Channel.

PROJECT STRUCTURE:
  app/src/main/java/com/example/
    di/           — Hilt modules
    data/
      local/      — Room database, DataStore
      remote/     — Retrofit API, interceptors
      repository/ — repository implementations
    domain/
      model/      — domain models
      usecase/    — use cases
    ui/
      feature/    — screens grouped by feature
        components/ — feature-specific composables
        FeatureScreen.kt
        FeatureViewModel.kt
      theme/      — Material theming
      navigation/ — NavHost and route definitions
    core/         — shared utilities
</architecture>

<jetpack_libraries>
UI:
- Jetpack Compose: declarative UI toolkit.
- Material 3: Google's design system for Compose.
- Navigation Compose: type-safe navigation with arguments.
- Accompanist: supplementary Compose libraries (permissions, system UI).

DATA:
- Room: SQLite abstraction with compile-time query verification.
- DataStore: key-value and typed data storage (replaces SharedPreferences).
- Retrofit: HTTP client for REST APIs with Kotlin coroutine support.
- Ktor: Kotlin-native HTTP client alternative.

ASYNC:
- Kotlin Coroutines: structured concurrency for async operations.
- Flow: reactive streams with backpressure (cold by default).
- StateFlow: observable state holder for ViewModels.
- WorkManager: guaranteed background work with constraints.

DEPENDENCY INJECTION:
- Hilt: compile-time DI built on Dagger. Google-recommended.
- Annotations: @HiltViewModel, @Inject, @Module, @Provides.
- Scoping: @Singleton, @ViewModelScoped, @ActivityScoped.
</jetpack_libraries>

<compose_patterns>
- Stateless composables: receive state and callbacks as parameters.
- State hoisting: lift state to the nearest common ancestor.
- Stable types: use @Stable or @Immutable for custom types to optimize recomposition.
- Remember: cache expensive computations within composition.
- LaunchedEffect: run suspend functions in composition scope.
- Slot-based design: use lambda parameters for composable customization.
- Theming: MaterialTheme for consistent design tokens.
</compose_patterns>
