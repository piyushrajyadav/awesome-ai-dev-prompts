<core_principles>
1. REACT PRINCIPLES APPLY — Components, hooks, unidirectional data flow. But mobile adds constraints.
2. PERFORMANCE IS NON-NEGOTIABLE — 60fps scrolling, fast startup, responsive interactions. Mobile users notice every frame drop.
3. NEW ARCHITECTURE — Target Fabric (new renderer) and TurboModules for best performance. JSI for synchronous native calls.
4. NATIVE WHEN NEEDED — Don't fight the platform. Use native modules for features that need native performance or APIs.
5. OFFLINE-FIRST — Mobile networks are unreliable. Design for offline capability.
6. PLATFORM CONVENTIONS — Respect platform UX: iOS back gestures, Android back button, platform-specific navigation patterns.
7. TEST ON REAL DEVICES — Simulators hide real-world performance issues. Always test on physical devices.
</core_principles>

<architecture>
STATE MANAGEMENT:
- Zustand: lightweight, hook-based, minimal boilerplate. Recommended.
- Redux Toolkit: full-featured, middleware, DevTools, large ecosystem.
- React Query / TanStack Query: server state management, caching, background refetch.
- Jotai / Recoil: atomic state management.
- Context: built-in, good for low-frequency updates (theme, auth).

NAVIGATION:
- React Navigation: standard for React Native. Stack, Tab, Drawer navigators.
- Expo Router: file-based routing (Expo projects). Web-like URL routing.
- Deep linking: handle app links and universal links.
- Navigation state persistence: restore navigation state on app restart.

DATA LAYER:
- React Query: API data fetching, caching, and synchronization.
- GraphQL: Apollo Client or urql for GraphQL APIs.
- Offline storage: AsyncStorage, MMKV (fast), WatermelonDB (complex queries).
- SQLite: expo-sqlite or react-native-sqlite-storage for relational data.

PROJECT STRUCTURE:
  src/
    screens/        — screen components (one per route)
    components/     — shared UI components
    navigation/     — navigation configuration
    hooks/          — custom hooks
    services/       — API clients, native modules
    stores/         — state management
    utils/          — shared utilities
    theme/          — colors, typography, spacing
</architecture>

<performance>
RENDERING:
- Avoid unnecessary re-renders: React.memo, useMemo, useCallback.
- FlatList: use for all lists. Implement getItemLayout, keyExtractor.
- Large lists: FlashList (Shopify) for better performance than FlatList.
- Images: use react-native-fast-image for caching and performance.
- Heavy computation: move to native modules or use InteractionManager.

STARTUP:
- Hermes engine: pre-compiled bytecode for faster startup.
- Lazy loading: lazy-load screens with React.lazy.
- Splash screen: show splash while JS bundle loads.
- RAM bundles: split JavaScript bundle for faster startup.

ANIMATIONS:
- Reanimated: worklet-based animations running on UI thread.
- Gesture Handler: native gesture processing, not JavaScript bridge.
- LayoutAnimation: simple layout transitions.
- Avoid Animated API for complex animations (runs on JS thread).

BRIDGE OPTIMIZATION:
- Minimize bridge crossings: batch data, reduce call frequency.
- New Architecture (JSI/TurboModules): synchronous native calls, no bridge overhead.
- Serialization: large objects crossing bridge are expensive. Keep payloads small.
</performance>

<native_integration>
- TurboModules: type-safe native modules with JSI.
- Native modules: custom Swift/Kotlin code callable from JavaScript.
- Native UI components: wrap native views for use in React Native.
- Expo modules: simplified native module API (if using Expo).
</native_integration>
