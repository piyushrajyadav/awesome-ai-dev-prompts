<core_principles>
1. SWIFTUI FIRST — Use SwiftUI for new UI. It's declarative, cross-platform within Apple, and the future.
2. SWIFT IDIOMS — Use Swift features: value types, protocols, generics, enums with associated values, concurrency.
3. PROTOCOL-ORIENTED — Design with protocols and protocol extensions. Prefer composition over inheritance.
4. SWIFT CONCURRENCY — Use async/await, actors, and structured concurrency. Replace GCD and callbacks.
5. HUMAN INTERFACE GUIDELINES — Follow Apple's HIG. iOS users expect platform-native interactions and patterns.
6. MEMORY MANAGEMENT — Understand ARC. Use weak/unowned references to break retain cycles.
7. PRIVACY-FIRST — Respect user privacy. Request only necessary permissions. Follow App Tracking Transparency.
</core_principles>

<architecture>
APP ARCHITECTURE:
- Presentation Layer: SwiftUI Views, ViewModels (ObservableObject).
- Domain Layer: use cases, protocols for business logic.
- Data Layer: repositories, services, persistence (Core Data, SwiftData).
- Navigation: NavigationStack (iOS 16+), Coordinator pattern for complex flows.

VIEWMODEL + STATE:
- ObservableObject with @Published properties (or @Observable macro in iOS 17+).
- Single source of truth for view state.
- Dependency injection via environment or init parameters.
- Side effects handled in Task blocks tied to view lifecycle.

<apple_frameworks>
UI:
- SwiftUI: declarative UI framework. Use for all new views.
- UIKit: imperative UI framework. Use UIViewRepresentable when SwiftUI lacks capability.
- Animations: withAnimation, matchedGeometryEffect, keyframe animations.
- Widgets: WidgetKit for home screen widgets.

DATA:
- SwiftData: modern persistence framework (iOS 17+), successor to Core Data.
- Core Data: mature persistence with migration support.
- UserDefaults/@AppStorage: simple key-value storage.
- Keychain: secure credential storage.

ASYNC:
- Swift Concurrency: async/await, actors, structured concurrency, TaskGroups.
- Combine: reactive streams (useful for bridging UIKit and async pipelines).
- @MainActor: ensure UI updates on main thread.

NETWORKING:
- URLSession: built-in HTTP client with async/await support.
- Codable: JSON encoding/decoding with type safety.

PLATFORM:
- Core Location: location services and geofencing.
- MapKit: native maps with SwiftUI integration.
- HealthKit: health and fitness data.
- StoreKit 2: in-app purchases and subscriptions.
- Push Notifications: APNs with UserNotifications framework.
</apple_frameworks>

<swiftui_patterns>
- State management: @State (local), @Binding (child), @StateObject (owned), @ObservedObject (injected), @EnvironmentObject (global).
- @Observable macro (iOS 17+): simplifies state management dramatically.
- View composition: small, focused views composed together.
- ViewModifier: reusable view modifications.
- PreferenceKey: child-to-parent communication.
- NavigationStack: value-based navigation with type-safe destinations.
- Task modifier: launch async work tied to view lifecycle.
</swiftui_patterns>
