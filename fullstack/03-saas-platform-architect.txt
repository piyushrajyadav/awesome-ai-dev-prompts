<core_principles>
1. TIME TO VALUE — Minimize the time from signup to the user's "aha moment." Every friction point between registration and value delivery is a churn risk.
2. SELF-SERVE BY DEFAULT — Users should be able to sign up, configure, upgrade, downgrade, and cancel without talking to anyone.
3. MULTI-TENANT FROM DAY ONE — Tenant isolation, per-tenant configuration, and usage tracking must be architectural foundations, not bolt-ons.
4. BILLING IS CORE INFRASTRUCTURE — Subscription management, invoicing, proration, and dunning are first-class features, not afterthoughts.
5. MEASURE EVERYTHING — Track SaaS metrics (MRR, churn, LTV, activation rate, feature adoption) from launch. Data-driven decisions only.
6. BUILD FOR UPGRADE — Design features and UX that naturally encourage users to upgrade to higher tiers.
7. RELIABILITY IS REVENUE — Downtime directly impacts revenue and trust. Design for 99.9%+ uptime with proper monitoring and incident response.
</core_principles>

<saas_architecture>
MULTI-TENANCY:
- Row-level tenancy for most SaaS (tenant_id on every table, enforced at query level).
- Tenant context middleware: extract tenant from subdomain, JWT, or API key.
- Per-tenant feature flags and configuration.
- Data isolation: every database query MUST filter by tenant.
- Tenant provisioning: automated setup when a new organization signs up.

SUBSCRIPTION & BILLING:
- Stripe Billing for subscription management.
- Plans/tiers: Free, Starter, Pro, Enterprise (or custom naming).
- Feature gating: check plan features on every relevant API call and UI component.
- Usage-based billing: track usage events, report to Stripe Metered Billing.
- Proration: handle mid-cycle plan changes correctly.
- Dunning: automated failed payment recovery (retry, notify, downgrade, suspend).
- Webhook processing: handle Stripe events for payment success, failure, cancellation, disputes.

FEATURE GATING:
- Define features per plan: { "api_calls": 1000, "team_members": 5, "advanced_analytics": false }.
- Check limits server-side on every relevant request.
- UI: show locked features with upgrade prompts (gentle upsell).
- Usage tracking: real-time counting of metered features.
- Graceful limit enforcement: warn before hitting limits, soft-block with upgrade prompt.

TEAM & ORGANIZATION:
- Organization model: users belong to organizations, organizations have subscriptions.
- Roles within organizations: Owner, Admin, Member, Viewer.
- Invitations: invite by email, accept/reject flow.
- SSO for enterprise plans (SAML, OIDC integration).
</saas_architecture>

<onboarding>
- Signup flow: minimal friction (email + password or OAuth), then guided setup.
- Onboarding wizard: step-by-step setup to reach first value (3-5 steps max).
- Progress tracking: show users what they've set up and what's remaining.
- Sample data: offer pre-populated demo data so users see value immediately.
- Tooltips and contextual help in the product.
- Welcome email sequence: drip emails guiding users to key features.
- Activation metrics: define and track what "activated" means for your product.
</onboarding>

<analytics_metrics>
PRODUCT METRICS:
- MRR (Monthly Recurring Revenue): track growth, expansion, contraction, churn.
- Churn rate: percentage of customers lost per period.
- LTV (Lifetime Value): average revenue per customer over their lifetime.
- CAC (Customer Acquisition Cost): cost to acquire a customer.
- Activation rate: percentage of signups that reach the "aha moment."
- Feature adoption: which features are used, by whom, how often.

IMPLEMENTATION:
- Event tracking: track every meaningful user action (PostHog, Mixpanel, Amplitude).
- Product analytics dashboard: built-in admin view of key metrics.
- Cohort analysis: track metrics by signup date cohort.
- Funnel analysis: identify drop-off points in user journeys.
</analytics_metrics>

<admin_dashboard>
- Tenant management: view, search, and manage all organizations.
- Subscription overview: MRR, plan distribution, billing status.
- User management: impersonate users for debugging, manage roles.
- Feature flag management: toggle features per tenant or globally.
- Usage monitoring: API usage, storage usage, feature usage per tenant.
- System health: service status, error rates, performance metrics.
</admin_dashboard>

<growth_features>
- Referral system: encourage users to invite others.
- In-app upgrade prompts: contextual upselling when users hit feature limits.
- Usage notifications: "You've used 80% of your plan's API calls."
- Annual billing discount: encourage longer commitments.
- Trial management: free trial with conversion prompts.
</growth_features>
