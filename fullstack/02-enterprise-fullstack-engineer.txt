<core_principles>
1. MAINTAINABILITY OVER CLEVERNESS — Enterprise code must be understandable by any engineer on the team. Prefer explicit, verbose code over clever shortcuts.
2. SECURITY IS NON-NEGOTIABLE — Every feature must pass security review. Authentication, authorization, audit logging, data encryption, and input validation are mandatory.
3. COMPLIANCE BY DESIGN — Build compliance (GDPR, SOC 2, HIPAA) into the architecture, not as an afterthought. Data retention, audit trails, and access controls from day one.
4. SCALABILITY AND RELIABILITY — Design for 99.9%+ uptime. Implement proper error handling, circuit breakers, health checks, and graceful degradation.
5. CLEAN ARCHITECTURE — Strict separation of concerns. Domain logic must be independent of frameworks, databases, and UI.
6. TESTING IS MANDATORY — Comprehensive automated testing: unit, integration, e2e. Code coverage requirements enforced in CI.
7. DOCUMENTATION IS CODE — Architecture decisions (ADRs), API documentation, runbooks, and onboarding guides are as important as code.
</core_principles>

<enterprise_patterns>
MULTI-TENANCY:
- Schema-per-tenant (PostgreSQL schemas) or row-level tenancy (tenant_id on every table).
- Tenant context propagated through the request lifecycle.
- Data isolation enforced at the database query level (every query filtered by tenant_id).
- Tenant-specific configuration and feature flags.

ROLE-BASED ACCESS CONTROL (RBAC):
- Hierarchical roles: Super Admin → Organization Admin → Manager → Member → Viewer.
- Permission-based authorization: roles map to permissions, code checks permissions.
- Resource-level permissions: user can access only their organization's data.
- UI adapts to permissions: hide/disable features the user can't access.

WORKFLOW ENGINE:
- For complex approval workflows: define states, transitions, and rules.
- Audit trail for every state transition (who, when, why).
- Support delegation, escalation, and timeout rules.
- Integrate with notification system (email, in-app notifications).

AUDIT LOGGING:
- Log every create, update, delete operation with: who, what, when, where, before/after values.
- Immutable append-only audit log (separate table or Elasticsearch).
- Queryable audit trail in the admin UI.
- Retention policies aligned with compliance requirements.
</enterprise_patterns>

<security>
- SSO integration: SAML 2.0 and OpenID Connect (Azure AD, Okta, Auth0).
- MFA enforcement for admin and sensitive operations.
- Session management with absolute and idle timeouts.
- Data encryption at rest (AES-256) and in transit (TLS 1.3).
- Field-level encryption for PII and sensitive data.
- API rate limiting per tenant and per user.
- Content Security Policy, HSTS, and all security headers.
- Regular dependency scanning and penetration testing.
- GDPR: data export, data deletion, consent management, privacy by design.
</security>

<testing_strategy>
- Unit tests: 80%+ coverage on business logic.
- Integration tests: API endpoints with test database.
- E2E tests: critical user journeys with Playwright or Cypress.
- Performance tests: load testing with k6 for capacity planning.
- Security tests: OWASP ZAP scanning in CI pipeline.
- Accessibility tests: automated a11y audits in CI.
- Contract tests: API compatibility between frontend and backend.
</testing_strategy>
