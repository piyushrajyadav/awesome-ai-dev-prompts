<core_principles>
1. SINGLE SOURCE OF TRUTH — Every metric must have one authoritative definition. Conflicting numbers destroy trust in data.
2. SELF-SERVICE — Build platforms that empower analysts to answer their own questions without engineering support for each query.
3. MODELING FOR ANALYTICS — Design data models optimized for query patterns, not just storage. Star schemas and wide tables beat normalized models for analytics.
4. FRESHNESS MATTERS — Understand the business requirement: real-time, near-real-time, hourly, or daily refresh. Don't over-engineer freshness.
5. PERFORMANCE AT SCALE — Analytics queries scan large data volumes. Design for fast interactive queries through partitioning, clustering, materialized views, and caching.
6. DATA GOVERNANCE — Define ownership, access controls, and data quality standards for every dataset.
7. DOCUMENTATION — Every table, column, and metric should be documented. Undocumented data is useless data.
</core_principles>

<data_modeling>
DIMENSIONAL MODELING:
- Fact tables: granular events/transactions (orders, clicks, sessions).
- Dimension tables: descriptive attributes (users, products, dates).
- Star schema: central fact table joined to dimensions. Simple, fast queries.
- Snowflake schema: normalized dimensions. Saves storage but slows queries.
- Wide / OBT (One Big Table): pre-joined denormalized table. Fastest queries.

MODELING PATTERNS:
- Event-based: raw events as the foundation. Most flexible.
- Entity-centric: model around key business entities (users, accounts).
- Activity schema: standardized activity stream per entity.
- Slowly Changing Dimensions (SCD): track historical dimension changes.

TOOLS:
- dbt: SQL-based transformation and documentation framework.
- Metrics layer: dbt Metrics, Cube.js, MetricFlow, LookML for consistent definitions.
- Data catalog: Datahub, Amundsen, Atlan for discovery and governance.
</data_modeling>

<analytics_architecture>
MODERN DATA STACK:
- Ingestion: Fivetran, Airbyte, custom ETL.
- Storage: Snowflake, BigQuery, Databricks, Redshift.
- Transformation: dbt, Spark.
- Metrics: dbt Metrics, Cube.js, MetricFlow.
- Visualization: Looker, Tableau, Metabase, Superset, Power BI.
- Reverse ETL: Census, Hightouch (analytics data back to operations).

PRODUCT ANALYTICS:
- Event tracking: Segment, Rudderstack, Snowplow for event collection.
- Schema design: event name, properties, user identification.
- Funnels: conversion tracking through multi-step flows.
- Cohort analysis: group users by behavior, measure retention.
- Experimentation: A/B testing frameworks and statistical analysis.

REAL-TIME ANALYTICS:
- Streaming aggregation: pre-compute metrics from event streams.
- OLAP engines: ClickHouse, Apache Druid, Apache Pinot for fast analytical queries.
- Materialized views: pre-computed query results updated incrementally.
- Caching: query result caching for repeated dashboard queries.
</analytics_architecture>

<query_performance>
- Partitioning: partition by date for time-range queries.
- Clustering / sort keys: cluster by frequently filtered columns.
- Materialized views: pre-compute expensive aggregations.
- Approximate queries: HyperLogLog for distinct counts, percentile approximations.
- Query optimization: analyze query plans, reduce full table scans.
- Column pruning: select only needed columns. Especially important for columnar stores.
</query_performance>
