================================================================================
PROMPT NAME: Multi-File Reasoning Framework
DESCRIPTION: Techniques for helping AI reason across multiple files to
understand complex interactions, data flows, and system behavior.
USE CASES:
  - Debugging issues spanning multiple files
  - Understanding feature implementations
  - Refactoring across file boundaries
  - Tracing data flow through system
================================================================================

<reasoning_strategies>
1. DEPENDENCY CHAIN TRACING:
   - Start from entry point
   - Follow imports/calls
   - Build execution path
   - Identify key decision points

2. DATA FLOW ANALYSIS:
   - Track data from source to destination
   - Identify transformations
   - Map state changes
   - Find data dependencies

3. CALL GRAPH CONSTRUCTION:
   - Map function calls across files
   - Identify call hierarchy
   - Find circular dependencies
   - Detect unused code

4. INTERFACE BOUNDARY ANALYSIS:
   - Identify module boundaries
   - Understand contracts between modules
   - Find coupling points
   - Detect abstraction leaks
</reasoning_strategies>

```
<task>

  - Purpose: [why]
  - Data transformed: [how]

  - Result: [what]

<key_files>
[Relevant code from each file in the flow]

```
</prompt_structure>

<analysis_techniques>
FORWARD TRACING (Top-Down):
- Start from user action or API call
- Follow execution path forward
- Track data transformations
- Identify final outcome

BACKWARD TRACING (Bottom-Up):
- Start from observed behavior or bug
- Work backwards to find cause
- Identify all paths that lead there
- Find root cause

BIDIRECTIONAL TRACING:
- Start from middle (suspected issue)
- Trace both forward and backward
- Understand full context
- Identify fix location

CROSS-CUTTING CONCERNS:
- Identify patterns across files
- Find common utilities
- Detect repeated logic
- Suggest refactoring opportunities
</analysis_techniques>
