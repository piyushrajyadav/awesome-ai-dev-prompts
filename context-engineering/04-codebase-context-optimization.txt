================================================================================
PROMPT NAME: Codebase Context Optimization
DESCRIPTION: Strategies for efficiently loading and managing codebase context
to maximize AI understanding while staying within token limits.
USE CASES:
  - Working with large codebases
  - Optimizing context for code generation
  - Reducing token usage while maintaining quality
  - Smart file selection for context
================================================================================

<context_selection_strategy>
PRIORITY LEVELS:

TIER 1 (Always Include):
- Files directly mentioned by user
- Current file being edited
- Main entry points (index.js, main.py)
- Core configuration files

TIER 2 (High Priority):
- Files imported by Tier 1 files
- Related test files
- Type definitions/interfaces
- Shared utilities used by target files

TIER 3 (Medium Priority):
- Files in same directory
- Similar files (by name/purpose)
- Recently modified files
- Documentation files

TIER 4 (Low Priority):
- Distant dependencies
- Generated files
- Third-party code
- Large data files
</context_selection_strategy>

<token_budget_management>
TYPICAL BUDGET ALLOCATION:
- System prompt: 2,000-5,000 tokens
- User instructions: 500-2,000 tokens
- Codebase context: 20,000-50,000 tokens
- Output buffer: 4,000-8,000 tokens

OPTIMIZATION TECHNIQUES:

1. SELECTIVE FILE LOADING:
   - Load only relevant functions, not entire files
   - Use AST parsing to extract specific classes/functions
   - Skip comments and docstrings if not needed

2. SUMMARIZATION:
   - Replace large files with structural summaries
   - Show function signatures instead of implementations
   - Provide file-level summaries for distant dependencies

3. COMPRESSION:
   - Remove whitespace and formatting
   - Minify when understanding doesn't require formatting
   - Use abbreviated variable names in examples

4. LAZY LOADING:
   - Start with minimal context
   - Request additional files only when needed
   - Iterative context expansion
</token_budget_management>

<file_relevance_scoring>
SCORING FACTORS:

Import Distance (40%):
- Direct import: 1.0
- 1 hop away: 0.7
- 2 hops away: 0.4
- 3+ hops away: 0.2

Recency (20%):
- Modified today: 1.0
- This week: 0.8
- This month: 0.5
- Older: 0.3

Name Similarity (15%):
- Exact match: 1.0
- Contains keyword: 0.7
- Related term: 0.4

Directory Proximity (15%):
- Same directory: 1.0
- Parent/child: 0.7
- Sibling: 0.5
- Distant: 0.2

File Type (10%):
- Source code: 1.0
- Tests: 0.8
- Config: 0.6
- Docs: 0.4

FINAL SCORE: Weighted sum of factors
</file_relevance_scoring>

<smart_filtering>
EXCLUDE FROM CONTEXT:
- node_modules, venv, vendor directories
- Build artifacts (dist, build, .next)
- Lock files (package-lock.json, yarn.lock)
- Binary files (images, videos, PDFs)
- Large data files (CSVs, JSONs > 100KB)
- Generated code (protobuf, GraphQL codegen)
- Minified files
- Log files

INCLUDE SELECTIVELY:
- package.json (dependencies section only)
- Config files (relevant sections)
- README (summary only)
- Tests (related to target code)
</smart_filtering>
