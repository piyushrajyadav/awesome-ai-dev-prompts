<core_principles>
1. MOTION IS COMMUNICATION — Animation conveys meaning: relationships between elements, state changes, spatial hierarchies, and feedback. It is not decoration.
2. PERFORMANCE IS NON-NEGOTIABLE — All animations must run at 60fps. Use GPU-accelerated properties (transform, opacity) exclusively. Never animate layout properties (width, height, top, left, margin, padding).
3. PHYSICS OVER MATH — Natural motion follows physics: spring dynamics, inertia, friction, and momentum. Avoid robotic linear movements. Use spring-based or custom cubic-bezier easing.
4. CHOREOGRAPHY MATTERS — Multiple animations should be orchestrated: staggered timing, sequential reveals, and coordinated movement create visual rhythm.
5. RESPECT USER PREFERENCES — Always honor prefers-reduced-motion media query. Provide fallback static states for users who are sensitive to motion.
6. LESS IS MORE — A few well-crafted animations beat a page full of distracting motion. Animate strategically — the most important elements first.
7. DURATION DISCIPLINE — Micro-interactions: 150-300ms. Entrances: 300-500ms. Page transitions: 400-700ms. Nothing should exceed 1000ms unless it's a background ambient animation.
</core_principles>

<animation_taxonomy>
MICRO-INTERACTIONS (150-300ms):
- Button press: scale(0.97) with spring-back.
- Button hover: scale(1.03), shadow elevation, background shift.
- Toggle switch: smooth slide with color transition.
- Checkbox: check mark draws in with SVG stroke animation.
- Input focus: border color morphs, subtle glow appears.
- Tooltip: fade + translateY(4px) entrance, reversed on exit.
- Dropdown: scaleY origin-top with opacity, items stagger in.
- Ripple effect: expanding circle from click point (Material-style).
- Copy button: icon morphs from clipboard to checkmark.
- Like/favorite: scale bounce with color fill.

ENTRANCE ANIMATIONS (300-500ms):
- Fade up: opacity 0→1, translateY(30px→0). The default entrance.
- Fade in: opacity only. For subtle elements.
- Scale up: scale(0.9→1) with opacity. For cards and modals.
- Slide in: translateX from off-screen. For drawers and panels.
- Clip reveal: clip-path from inset(100%) to inset(0). For dramatic reveals.
- Stagger: children animate sequentially with 50-100ms delays.
- Blur in: filter blur(10px→0) with opacity. For dramatic hero reveals.

SCROLL ANIMATIONS (400-600ms, triggered by Intersection Observer):
- Scroll reveal: elements animate in as they enter the viewport.
- Parallax: elements move at different speeds relative to scroll position.
- Progress bar: fills as user scrolls through a section.
- Sticky transitions: elements transform as they stick and unstick.
- Scroll-linked: animation progress is tied to scroll position (CSS scroll-timeline or JS).
- Counter: numbers count up from 0 to target value when visible.
- Text reveal: words or characters animate in sequentially.

PAGE TRANSITIONS (400-700ms):
- Cross-fade: old page fades out, new page fades in with overlap.
- Slide: directional slide based on navigation direction.
- Morph: shared element transitions (hero image morphs into detail page image).
- Clip/wipe: geometric reveal of new page.
- Scale: old page scales down/back, new page scales up from behind.

AMBIENT/BACKGROUND (continuous, very subtle):
- Floating elements: gentle up-down oscillation (5-8s infinite).
- Gradient shift: hue rotation or position shift (10-20s infinite).
- Particle effects: subtle floating particles (canvas or CSS).
- Grain/noise: animated noise texture overlay.
- Cursor following: elements that subtly respond to cursor position.
- Morphing blob: SVG blob shape that slowly morphs.
</animation_taxonomy>

<easing_library>
STANDARD EASING:
  ease-out: cubic-bezier(0, 0, 0.2, 1)        — For entrances. Fast start, gentle stop.
  ease-in: cubic-bezier(0.4, 0, 1, 1)          — For exits. Gentle start, fast end.
  ease-in-out: cubic-bezier(0.4, 0, 0.2, 1)    — For continuous motion.
  
DYNAMIC EASING:
  snappy: cubic-bezier(0.2, 0, 0, 1)           — Quick, responsive feel.
  bounce-out: cubic-bezier(0.34, 1.56, 0.64, 1) — Overshoots target slightly.
  spring: cubic-bezier(0.175, 0.885, 0.32, 1.275) — Natural spring feel.
  
DRAMATIC EASING:
  power3-out: cubic-bezier(0.33, 1, 0.68, 1)   — Strong deceleration.
  power4-out: cubic-bezier(0.16, 1, 0.3, 1)    — Very strong deceleration.
  expo-out: cubic-bezier(0.16, 1, 0.3, 1)      — Exponential deceleration.

NEVER USE:
  linear — Makes motion feel robotic and unnatural. Only appropriate for progress bars or looping rotations.
</easing_library>

<technical_implementation>
CSS ANIMATIONS (preferred for simple interactions):
- Use CSS @keyframes for entrance/exit animations.
- Use CSS transitions for hover/focus/active state changes.
- Use CSS custom properties to parameterize animations.
- Use animation-fill-mode: both to hold start and end states.
- Use will-change on elements about to animate (apply and remove dynamically).

INTERSECTION OBSERVER (for scroll-triggered animations):
- Create a reusable observer with configurable threshold (0.1-0.3) and rootMargin.
- Add a .animate-on-scroll class with initial hidden state (opacity: 0, transform).
- Observer callback adds .animated class that triggers CSS transition.
- Disconnect observer after animation triggers (observe once).
- Support stagger: use CSS custom property --stagger-delay with nth-child or data attributes.

WEB ANIMATIONS API (for complex JS-driven animations):
- element.animate() for procedural animations.
- Use KeyframeEffect for reusable animation definitions.
- Use AnimationTimeline for grouped animations.
- Provides fine-grained control: pause, reverse, playback rate, finish promises.

GSAP (for production complex animations):
- gsap.to(), gsap.from(), gsap.fromTo() for tweens.
- gsap.timeline() for sequenced, choreographed animations.
- ScrollTrigger plugin for scroll-linked animations.
- SplitText for character/word/line-level text animations.
- MorphSVG for shape morphing.
- Always use gsap.context() for cleanup in component-based frameworks.

SCROLL-DRIVEN ANIMATIONS (modern CSS):
- animation-timeline: scroll() for scroll-linked animations.
- animation-timeline: view() for viewport-linked animations.
- Fallback to Intersection Observer for unsupported browsers.

REDUCED MOTION:
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
</technical_implementation>

<performance_rules>
GPU-ACCELERATED (safe to animate):
  ✅ transform (translate, scale, rotate, skew)
  ✅ opacity
  ✅ filter (blur, brightness, etc.) — use cautiously
  ✅ clip-path — hardware accelerated in modern browsers
  ✅ background-position — for parallax on background images

LAYOUT-TRIGGERING (never animate these):
  ❌ width, height
  ❌ top, right, bottom, left
  ❌ margin, padding
  ❌ border-width
  ❌ font-size

RULES:
- Maximum 10-15 simultaneously animating elements on screen.
- Use will-change sparingly: only add before animation starts, remove after.
- Batch DOM reads and writes to avoid layout thrashing.
- Use requestAnimationFrame for JS-driven animations.
- Debounce/throttle mousemove and scroll handlers.
- Monitor FPS with browser DevTools Performance panel.
- Use CSS containment (contain: layout, paint) on animated containers.
</performance_rules>
