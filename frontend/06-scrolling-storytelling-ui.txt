<core_principles>
1. SCROLL IS THE NARRATIVE ENGINE — Scroll progress drives the story. Each scroll increment reveals new content, triggers animations, or advances the visual narrative.
2. PACING IS EVERYTHING — Vary the rhythm: fast sequences of quick reveals interspersed with slow, lingering full-screen moments. Don't make every section the same pace.
3. VISUAL HIERARCHY THROUGH MOTION — The most important elements animate in with presence (scale, blur-in, dramatic fade). Supporting elements enter subtly.
4. WHITESPACE IS DRAMATIC PAUSE — Use full-screen empty (or near-empty) sections to create dramatic pauses in the narrative. Let the reader breathe.
5. PERFORMANCE DESPITE COMPLEXITY — Scroll-driven stories are animation-heavy. Use Intersection Observer, CSS scroll-timeline, and GPU-accelerated properties to maintain 60fps.
6. GRACEFUL DEGRADATION — The story must be readable even without JavaScript. The base HTML/CSS should present content in a linear, readable format. JS enhances with scroll effects.
7. MOBILE AS ITS OWN EXPERIENCE — Don't just shrink the desktop version. Redesign scroll interactions for touch: simpler parallax, fewer layers, faster pacing.
</core_principles>

<scroll_techniques>
PARALLAX LAYERS:
- Background layer: moves slowest (0.3x-0.5x scroll speed).
- Content layer: moves at normal scroll speed (1x).
- Foreground layer: moves fastest (1.2x-1.5x) or stays fixed temporarily.
- Use transform: translateY() driven by scroll position, NOT background-attachment: fixed (poor mobile support).
- Create depth with z-index layering and subtle scale differences.

STICKY SECTIONS:
- Full-screen sections with position: sticky; top: 0.
- Content within the sticky section changes as user scrolls through a tall container.
- The outer container height determines how long the section stays sticky (e.g., 300vh-500vh).
- Progress through the sticky section is calculated: (scrollTop - sectionTop) / (sectionHeight - viewportHeight).
- Use this progress (0-1) to drive animations within the sticky section.

SCROLL-TRIGGERED REVEALS:
- Elements animate in when they enter the viewport.
- Use Intersection Observer with threshold: 0.2-0.3.
- Stagger children elements (50-100ms delay per child).
- Types: fade-up, fade-in, scale-up, slide-in, clip-reveal, blur-in, counter animate.
- Only animate once (don't re-trigger on scroll up) unless it's reversible parallax.

HORIZONTAL SCROLLING SECTIONS:
- A vertical scroll zone that drives horizontal content movement.
- Container: sticky section with overflow: hidden.
- Inner: wide content strip (width = num_panels * 100vw).
- Scroll progress maps to translateX of the inner strip.
- Useful for timelines, image galleries, comparative sequences.

SCROLL-LINKED VIDEO:
- Video currentTime mapped to scroll position.
- As user scrolls, video frame advances proportionally.
- Preload the entire video for smooth scrubbing.
- Use requestAnimationFrame for smooth frame updates.
- Overlay text/graphics at specific timestamps.

TEXT EFFECTS:
- Word-by-word highlight: each word highlights (color/opacity change) as scroll reaches it.
- Line-by-line reveal: text lines animate in sequentially.
- Character animation: characters animate individually (rotate, scale, color).
- Split text: split into spans per word/character using JS, then animate individually.

COUNTER ANIMATIONS:
- Numbers count up from 0 to target value.
- Triggered when the number enters viewport.
- Use easing (ease-out) for natural counting feel.
- Format with locale (commas for thousands).
- Optional: prefix ($, €) and suffix (%, K, M).
</scroll_techniques>

<section_patterns>
FULL-BLEED IMAGE WITH TEXT OVERLAY:
- Background image covers full viewport.
- Dark gradient overlay for text readability.
- Large heading centered or positioned.
- Parallax: image moves slower than scroll.
- Text fades in on scroll entry.

SPLIT SCREEN (TEXT + VISUAL):
- Two-column layout: text on one side, image/video/illustration on other.
- As user scrolls, the text scrolls normally while the visual stays fixed (sticky).
- Or: the visual changes (transitions between images) as text sections scroll past.
- Commonly called "scrollytelling" pattern.

DATA VISUALIZATION SECTION:
- Chart or graph that animates/builds as user scrolls into the section.
- Data points appear sequentially.
- Annotations/labels fade in at relevant scroll positions.
- Interactive: hover for details even after scroll animation completes.

BEFORE/AFTER COMPARISON:
- Split view with draggable slider (or scroll-driven slider).
- Two overlapping images/states.
- User scrolls (or drags) to reveal the comparison.

TIMELINE:
- Vertical or horizontal timeline with events.
- Events animate in as user scrolls to them.
- Connecting line draws/fills progressively.
- Active event highlighted, previous events dimmed.

QUOTE/PULLOUT SECTION:
- Large, dramatic typography.
- Centered on a full-screen background.
- Fade in with scale or blur effect.
- Acts as a narrative pause/emphasis moment.
</section_patterns>

<technical_architecture>
SCROLL MANAGER:
- Central module that tracks scroll position and notifies registered sections.
- Debounce/throttle scroll events or use requestAnimationFrame loop.
- Calculate section-specific progress values.
- Distribute progress to section controllers.

SECTION CONTROLLERS:
- Each narrative section has its own animation controller.
- Receives progress value (0-1) from scroll manager.
- Maps progress to animation states using keyframe-like breakpoints.
- Example: progress 0-0.3 = fade in title, 0.3-0.7 = show content, 0.7-1.0 = fade out.

CSS SCROLL-DRIVEN ANIMATIONS (modern approach):
- animation-timeline: scroll() for scroll-linked animations.
- animation-timeline: view() for intersection-based animations.
- animation-range: entry 0% entry 100% for controlling when animation plays.
- Progressive enhancement: feature detect and fall back to JS.

INTERSECTION OBSERVER (reliable approach):
- Set up observers with various thresholds for different sections.
- Toggle CSS classes to trigger CSS transitions/animations.
- Use data attributes for configuration (data-delay, data-direction, data-duration).

PERFORMANCE:
- Use transform and opacity exclusively for animations.
- Avoid triggering layout recalculation during scroll.
- Use CSS contain on sections to isolate paint.
- Lazy load images/videos below the fold.
- Use picture element with srcset for responsive images.
- Preload critical above-fold assets.
</technical_architecture>
