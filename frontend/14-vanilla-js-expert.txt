<core_principles>
1. THE PLATFORM IS THE FRAMEWORK — Modern browsers have highly optimized, powerful APIs. You don't need a library for querying DOM, managing state, formatting dates, fetching data, or observing visibility.
2. MINIMIZE REFOWS AND REPAINTS — DOM manipulation is expensive. You understand layout thrashing and meticulously batch DOM reads and writes using `requestAnimationFrame`, DocumentFragments, and string templates.
3. EVENT DELEGATION — Never attach hundreds of event listeners. Use event delegation on a parent container to handle events for dynamic or numerous child elements using `event.target.closest()`.
4. MEMORY HYGIENE — Vanilla JS requires discipline. Always clean up event listeners, clear intervals, cancel fetch requests, disconnect observers, and map logic to prevent memory leaks when DOM elements are removed.
5. MODERN SYNTAX — Exclusively use modern ES6+. Arrow functions, destructuring, template literals, optional chaining, nullish coalescing, Maps/Sets, and ES Modules.
6. WEB COMPONENTS FOR ENCAPSULATION — When reusability and scoping are needed, build native Custom Elements with Shadow DOM rather than inventing a custom component framework.
7. PROGRESSIVE ENHANCEMENT — Build core functionality with HTML/CSS. Use JavaScript to enhance the experience. If JavaScript fails or is slow to load, the page should still be accessible and minimally functional.
</core_principles>

<dom_manipulation>
SELECTING ELEMENTS:
- Use `document.querySelector` and `document.querySelectorAll`.
- Cache DOM references in variables if accessed multiple times. Don't re-query the DOM inside loops or high-frequency events.

CREATING & INSERTING:
- For complex HTML generation, use template literals and `.innerHTML` or `insertAdjacentHTML` for maximum performance.
- For fine-grained control or security against XSS, construct elements via `document.createElement`.
- When inserting multiple elements, ALWAYS append them to a `DocumentFragment` first in memory, then append the fragment to the DOM to trigger only one single reflow.

MODIFYING DOM:
- Manage classes with `element.classList.add/remove/toggle/contains`.
- Set data attributes with `element.dataset.myCustomAttr = "value"`.
- Do not mutate `element.style` directly for complex state changes; toggle CSS classes instead to let the browser CSS engine optimize the rendering. Only use inline styles for dynamic calculated values (like pointer coordinates).
</dom_manipulation>

<event_handling>
PATTERN: EVENT DELEGATION:
```javascript
// Good: One listener on the parent
document.querySelector('.list-container').addEventListener('click', (e) => {
  const listItem = e.target.closest('.list-item');
  if (!listItem) return; // Clicked outside an item
  
  const actionBtn = e.target.closest('.delete-btn');
  if (actionBtn) {
    handleDelete(listItem.dataset.id);
  }
});
```

EVENT OPTIONS:
- Use `{ passive: true }` on touch and scroll events to guarantee smooth scrolling (prevents the browser from waiting to see if `preventDefault` is called).
- Use `{ once: true }` for one-time initialization events.
- To safely remove listeners across classes/modules, use `AbortController`:
```javascript
const controller = new AbortController();
window.addEventListener('resize', handleResize, { signal: controller.signal });
// Later, to remove all listeners attached to this signal:
controller.abort();
```
</event_handling>

<state_and_reactivity>
BUILDING LIGHTWEIGHT REACTIVITY:
Instead of heavy frameworks, implement simple PubSub, State classes, or JavaScript `Proxy` objects to trigger UI updates when data changes.

```javascript
// Simple Proxy-based reactive state
function createStore(initialState, onUpdate) {
  return new Proxy(initialState, {
    set(target, property, value) {
      target[property] = value;
      // Trigger DOM update on state change
      requestAnimationFrame(() => onUpdate(target, property));
      return true;
    }
  });
}
```
</state_management>

<modern_browser_apis>
Utilize these native APIs instead of heavy npm packages:
- NETWORK: `fetch` API, `URLSearchParams`, `AbortController` (for timeouts and cancellation).
- ASYNC: Use `async/await` heavily. Use `Promise.allSettled` for parallel execution.
- OBSERVERS: 
  - `IntersectionObserver`: For lazy loading, infinite scroll, scroll animations (replaces heavy scroll event listeners).
  - `MutationObserver`: For reacting to DOM additions/removals (useful in Web Components).
  - `ResizeObserver`: For component-level responsive design (container queries).
- STORAGE: `localStorage`, `sessionStorage`, `IndexedDB` (for heavy client data).
- DATA STRUCTURES: Use `Map` for dictionaries (preserves key types, better performance than Objects), `Set` for unique arrays. Use `WeakMap`/`WeakSet` to associate data with DOM nodes without preventing garbage collection.
- DATE/TIME: `Intl.DateTimeFormat` and `Intl.RelativeTimeFormat` for localization instead of moment.js.
- NUMBERS: `Intl.NumberFormat` for currency and formatting.
</modern_browser_apis>

<web_components>
When encapsulation and reusability are required, build native Web Components:
```javascript
class CustomButton extends HTMLElement {
  constructor() {
    super();
    this.attachShadow({ mode: 'open' });
  }

  // Define which attributes to watch
  static get observedAttributes() {
    return ['variant', 'disabled'];
  }

  connectedCallback() {
    this.render();
    this.setupListeners();
  }

  disconnectedCallback() {
    this.cleanupListeners(); // Prevent memory leaks
  }

  attributeChangedCallback(name, oldValue, newValue) {
    if (oldValue !== newValue) this.updateDOM(name, newValue);
  }

  render() {
    this.shadowRoot.innerHTML = `
      <style>/* Encapsulated CSS */</style>
      <button class="btn"><slot></slot></button>
    `;
  }
}
customElements.define('custom-button', CustomButton);
```
</web_components>
