<core_principles>
1. RELIABILITY OVER AUTONOMY — A reliable agent that completes 95% of tasks correctly beats an ambitious agent that fails unpredictably. Start simple, add complexity gradually.
2. TOOL DESIGN IS CRITICAL — The quality of tools (clear descriptions, good error messages, proper input validation) determines agent success more than the LLM's reasoning ability.
3. OBSERVE EVERYTHING — Log every agent step: reasoning, tool calls, observations, decisions. You cannot debug what you cannot see.
4. BOUNDED EXECUTION — Always set maximum step limits, token budgets, and timeout constraints. Unbounded agents are dangerous and expensive.
5. FAIL GRACEFULLY — Agents will make mistakes. Design recovery strategies: retry, ask for help, or return partial results with explanation.
6. HUMAN IN THE LOOP — For high-stakes actions, require human approval. Not everything should be autonomous.
7. COST CONTROL — Agent loops consume many API calls. Monitor and budget token usage per task.
</core_principles>

<agent_architectures>
REACT (REASON + ACT):
- Loop: Think → Act (use tool) → Observe (tool result) → Think → ...
- Best for: straightforward tool-use tasks with clear goals.
- Simple to implement, easy to debug.
- Risk: can get stuck in loops or repeat actions.

PLAN-AND-EXECUTE:
- Phase 1: Create a step-by-step plan for the task.
- Phase 2: Execute each step, potentially re-planning after observations.
- Best for: complex multi-step tasks where upfront planning helps.
- More reliable for long-horizon tasks than pure ReAct.

TREE OF THOUGHT / REFLECTION:
- Generate multiple reasoning paths, evaluate them, select the best.
- Self-reflection: agent critiques its own output and iterates.
- Best for: tasks requiring creativity or complex reasoning.
- More expensive but higher quality for difficult problems.

MULTI-AGENT SYSTEMS:
- Specialized agents collaborating on a task (researcher, coder, reviewer).
- Orchestrator agent delegates to and coordinates specialist agents.
- Communication protocols between agents.
- Best for: complex tasks requiring diverse capabilities.
- Frameworks: AutoGen, CrewAI, LangGraph.
</agent_architectures>

<tool_design>
PRINCIPLES:
- Clear, descriptive tool names and descriptions (the LLM reads these).
- Well-defined input schemas with types and descriptions.
- Informative error messages that help the agent recover.
- Idempotent tools where possible (safe to retry).
- Atomic operations (one clear action per tool).

COMMON TOOLS:
- Web search: search the internet for information.
- Web browse: read the content of a specific URL.
- Code execution: run code in a sandboxed environment.
- File operations: read, write, list files.
- API calls: interact with external services.
- Database queries: search structured data.
- Calculator: precise mathematical computations.

SAFETY:
- Sandbox code execution (Docker containers, E2B, Modal).
- Read-only tools by default, write tools require explicit permission.
- Input validation to prevent injection attacks.
- Rate limiting on expensive or destructive operations.
</tool_design>

<memory_systems>
SHORT-TERM MEMORY:
- Conversation history: recent agent steps and observations.
- Working memory: current task state and intermediate results.
- Context window management: summarize old steps to fit context limit.

LONG-TERM MEMORY:
- Vector store: semantic search over past experiences and knowledge.
- Episodic memory: store successful task completions for future reference.
- Entity memory: track facts about entities encountered.

MEMORY MANAGEMENT:
- Summarize long conversation histories to prevent context overflow.
- Prioritize recent and relevant information in context.
- Persist memory across sessions for stateful agents.
</memory_systems>

<evaluation>
- Task completion rate: percentage of tasks successfully completed.
- Step efficiency: average number of steps to complete a task.
- Error recovery rate: how often the agent recovers from mistakes.
- Cost per task: total API token cost per completed task.
- Safety: rate of harmful or unauthorized actions.
- Qualitative evaluation: human review of agent reasoning traces.
</evaluation>
