<core_principles>
1. AUTHENTICATE EVERY REQUEST — No anonymous API access unless explicitly designed. Validate tokens on every request.
2. AUTHORIZE AT THE OBJECT LEVEL — Check that the authenticated user has access to the specific resource, not just the endpoint.
3. VALIDATE ALL INPUT — Schema validation, type checking, range validation on every parameter. Reject invalid requests.
4. RATE LIMIT EVERYTHING — Every endpoint has a rate limit. Prevent abuse, brute force, and resource exhaustion.
5. MINIMIZE DATA EXPOSURE — Return only the data the client needs. Never expose internal IDs, metadata, or debug info.
6. VERSIONING AND DEPRECATION — Version APIs explicitly. Deprecate old versions with clear migration paths.
7. ZERO TRUST — Validate every request as if it could be malicious. Don't trust client-side validation.
</core_principles>

<owasp_api_security>
API1 BROKEN OBJECT LEVEL AUTHORIZATION (BOLA/IDOR):
- Verify the user owns/has access to the requested resource.
- Don't rely on client-supplied resource IDs without authorization checks.
- Use UUIDs instead of sequential IDs to prevent enumeration.
- Authorization check on every data access, not just the endpoint.

API2 BROKEN AUTHENTICATION:
- Use standard protocols: OAuth 2.0, OpenID Connect.
- Validate JWTs: signature, expiration, issuer, audience.
- Rate limit authentication endpoints aggressively.
- Account lockout after failed attempts.

API3 BROKEN OBJECT PROPERTY LEVEL AUTHORIZATION:
- Don't expose all object properties to all users.
- Mass assignment protection: explicit allowlists for writable fields.
- Different response schemas for different user roles.

API4 UNRESTRICTED RESOURCE CONSUMPTION:
- Rate limiting per user, per IP, per endpoint.
- Request size limits: body size, query parameter count.
- Pagination: enforce maximum page sizes.
- Timeout: limit execution time for expensive operations.
- Cost-based rate limiting: charge more for expensive operations.

API5 BROKEN FUNCTION LEVEL AUTHORIZATION:
- Admin endpoints must verify admin role.
- Separate admin and user API surfaces when possible.
- Don't rely on path obscurity for security.

API6 UNRESTRICTED ACCESS TO SENSITIVE BUSINESS FLOWS:
- Rate limit sensitive operations (purchase, transfer, registration).
- CAPTCHA or proof-of-work for bot-susceptible flows.
- Device fingerprinting for abuse detection.

API7 SERVER SIDE REQUEST FORGERY (SSRF):
- Validate and restrict user-supplied URLs.
- Block internal network ranges and cloud metadata endpoints.
- Use allowlists for permitted external domains.

API8 SECURITY MISCONFIGURATION:
- Disable unnecessary HTTP methods (TRACE, OPTIONS if not needed).
- Remove default error pages and stack traces.
- CORS: restrict allowed origins. Never use wildcard with credentials.
- Security headers on all API responses.

API9 IMPROPER INVENTORY MANAGEMENT:
- Maintain API inventory: all endpoints, versions, and environments.
- Decommission unused APIs and endpoints.
- API documentation as security documentation.

API10 UNSAFE CONSUMPTION OF APIS:
- Validate responses from third-party APIs.
- Implement timeouts and circuit breakers for external calls.
- Don't trust external API data — validate before use.
</owasp_api_security>

<api_security_controls>
API GATEWAY:
- Centralized authentication and authorization enforcement.
- Rate limiting and throttling.
- Request/response validation against OpenAPI schema.
- IP allowlisting/blocklisting.
- WAF integration for attack detection.
- Logging and monitoring of all API traffic.

INPUT VALIDATION:
- OpenAPI/JSON Schema validation for request bodies.
- Query parameter type and range validation.
- Path parameter validation (UUIDs, expected formats).
- Content-Type enforcement: reject unexpected content types.
- Request size limits at network and application layers.

OUTPUT SECURITY:
- Consistent error responses: don't leak implementation details.
- Field filtering: return only requested/authorized fields.
- Pagination: prevent unbounded data retrieval.
- Sensitive data masking in logs and error responses.
</api_security_controls>
