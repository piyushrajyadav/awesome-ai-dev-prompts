================================================================================
PROMPT NAME: Code Quality Evaluation Framework
DESCRIPTION: Comprehensive framework for evaluating AI-generated code quality
across multiple dimensions including correctness, maintainability, performance,
security, and adherence to best practices.
USE CASES:
  - Evaluating code generated by different prompts
  - Comparing prompt versions (A/B testing)
  - Establishing code quality baselines
  - Automated code review scoring
================================================================================

<evaluation_dimensions>
1. CORRECTNESS (Weight: 30%)
   - Functional requirements met
   - Edge cases handled
   - Error handling implemented
   - Type safety maintained
   - Logic correctness

2. MAINTAINABILITY (Weight: 25%)
   - Code readability
   - Clear naming conventions
   - Appropriate comments
   - Modular structure
   - DRY principle followed

3. PERFORMANCE (Weight: 15%)
   - Algorithm efficiency
   - Resource usage
   - Scalability considerations
   - No obvious bottlenecks

4. SECURITY (Weight: 15%)
   - Input validation
   - No injection vulnerabilities
   - Secure authentication/authorization
   - Sensitive data handling
   - Dependency security

5. BEST PRACTICES (Weight: 15%)
   - Language idioms followed
   - Framework conventions respected
   - Testing included
   - Documentation present
   - Error messages helpful
</evaluation_dimensions>

<scoring_rubric>
CORRECTNESS SCORING (0-10):
10: Perfect implementation, all requirements met, all edge cases handled
8-9: Fully functional, minor edge cases missing
6-7: Core functionality works, some requirements incomplete
4-5: Partially functional, significant gaps
2-3: Major functionality broken
0-1: Does not work at all

MAINTAINABILITY SCORING (0-10):
10: Exceptionally clean, self-documenting, perfect structure
8-9: Very readable, well-organized, good naming
6-7: Readable with effort, acceptable structure
4-5: Confusing in places, poor organization
2-3: Hard to understand, messy structure
0-1: Unmaintainable spaghetti code

PERFORMANCE SCORING (0-10):
10: Optimal algorithms, excellent resource usage
8-9: Efficient implementation, no obvious issues
6-7: Acceptable performance, minor inefficiencies
4-5: Noticeable performance issues
2-3: Significant performance problems
0-1: Completely inefficient, unusable at scale

SECURITY SCORING (0-10):
10: Comprehensive security, defense in depth
8-9: Strong security, minor gaps
6-7: Basic security present, some vulnerabilities
4-5: Multiple security issues
2-3: Critical vulnerabilities present
0-1: Completely insecure

BEST PRACTICES SCORING (0-10):
10: Exemplary adherence to all standards
8-9: Follows most best practices
6-7: Some best practices followed
4-5: Many best practices ignored
2-3: Poor adherence to standards
0-1: No best practices followed
</scoring_rubric>

<evaluation_process>
STEP 1: AUTOMATED CHECKS
- Run linters (ESLint, Pylint, RuboCop, etc.)
- Run static analysis tools (SonarQube, CodeQL)
- Check test coverage
- Run security scanners (Snyk, Bandit)
- Measure cyclomatic complexity
- Check for code smells

STEP 2: FUNCTIONAL TESTING
- Execute unit tests
- Run integration tests
- Test edge cases manually
- Verify error handling
- Check performance benchmarks

STEP 3: MANUAL REVIEW
- Read code for clarity
- Assess architecture decisions
- Evaluate naming and structure
- Check documentation quality
- Review security considerations

STEP 4: CALCULATE SCORES
- Score each dimension (0-10)
- Apply weights to get final score
- Document specific issues found
- Provide improvement recommendations
</evaluation_process>

<comparison_framework>
When comparing multiple code versions:

1. Run same evaluation on all versions
2. Compare scores across dimensions
3. Identify which version excels in each area
4. Calculate statistical significance if multiple samples
5. Consider trade-offs (e.g., performance vs maintainability)
6. Recommend best version or hybrid approach
</comparison_framework>
